🚀How to Make Your First Payment (5 Minutes)

1Start the Application

Open PowerShell in your project folder and run:

cd MyBlazorApp dotnet run

Wait for the message: Now listening on: https://localhost:7000

2Open the Application

Navigate to your browser:

https://localhost:7000

3Go to Stripe Payment

In the left menu, click "Stripe Payment"

Or go directly to: https://localhost:7000/stripe-checkout

4Fill the Form

You'll see a simple form:

  • Product Name: "Sample Product" (or customize)
  • Amount ($): 29.99 (or customize)

5Click "Proceed to Payment"

You'll see a spinner saying "Processing payment..."

Your backend is creating a Stripe session.

6Stripe Checkout Opens

You'll be redirected to Stripe's hosted checkout page.

Fill in the payment details below.

7See Results

✅ Success: You'll see "Payment Successful!"

❌ Cancelled: You'll see "Payment Cancelled"

🐛Troubleshooting

❌ Problem: "Stripe is not defined"

Solution:

  • Open browser console (F12)
  • Reload the page (Ctrl+R)
  • Check for red errors
❌ Problem: "Session ID is invalid"

Solution:

  • Check appsettings.json SecretKey
  • Must start with sk_test_
  • No extra spaces
❌ Problem: "Publishable Key is not configured"

Solution:

  • Check appsettings.json PublishableKey
  • Must start with pk_test_
  • Cannot be empty
💡 Debug Tip: Open F12 (Browser Console) to see detailed logs of what's happening during payment processing.

📊Verify Payment in Stripe Dashboard

View Your Test Payments

  1. Go to https://dashboard.stripe.com
  2. Sign in with your account
  3. Click "Payments" in the menu
  4. You'll see all your test transactions
  5. Click any payment to see details

⚠️Important Notes

⚠️ Never use real credit cards in test mode!

Test mode only accepts the test card numbers listed above. Real cards will be rejected.

🔒 Never share your Secret Key!

Your Secret Key should only be in appsettings.json on the server. Never in JavaScript or frontend code.

🚀 For Production:

Replace test keys with live keys (starting with pk_live_ and sk_live_). Ensure HTTPS is enabled.
