┌────────────────────────────────────────────────────────────────────────────┐
│  🎯 CASOS DE USO                                                           │
└────────────────────────────────────────────────────────────────────────────┘

  1. Usuario Nuevo:
     Accede → Ve /login → Click "Create Account" → Va a /register
     → Ingresa datos → Se registra → Va a /login con creds nuevas
     → Authentica → Ve dashboard

  2. Usuario Existente:
     Accede → Automáticamente en /login → Ingresa creds
     → Authentica → Ve dashboard con sidebar → Puede navegar

  3. Hacer Pago:
     Dashboard → Click "New Payment" → Va a /stripe-checkout
     → Ingresa datos → Click "Proceed to Payment"
     → Redirige a Stripe → Completa pago
     → Vuelve a /payment-success (✅) o /payment-cancel (❌)

  4. Ver Información:
     Dashboard → Click "Information" → Va a /stripe-info
     → Lee documentación de Stripe/Blazor

  5. Logout:
     Desde cualquier página → Click "Logout" en sidebar
     → Sesión se limpia → Redirige a /login
     → Menú desaparece


┌────────────────────────────────────────────────────────────────────────────┐
│  🆘 TROUBLESHOOTING                                                        │
└────────────────────────────────────────────────────────────────────────────┘

  ❌ "Unable to connect to MySQL"
     → Verificar que MySQL está corriendo: mysql -u root -p root -e "SELECT 1"
  
  ❌ "Invalid credentials" en login
     → Ejecutar SEED_DEMO_USERS.sql para recrear usuarios
  
  ❌ "Port already in use"
     → Usar otro puerto: dotnet run --urls "http://localhost:5092"
  
  ❌ "Build failed"
     → dotnet clean && dotnet build
  
  ❌ "Migrations failed"
     → dotnet ef database drop && dotnet ef database update
